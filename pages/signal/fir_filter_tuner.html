FIR 滤波器参数调节器

拖动阶数、截止频率和窗函数，观察“平滑 ↔ 延迟 ↔ 计算量”的取舍

FIR 的核心思想是：当前输出只由当前输入和过去有限个输入共同决定，不使用历史输出反馈。 它可以做最简单的滑动平均，也可以通过 sinc + 窗函数设计成更标准的低通滤波器。 在 MCU 和工业传感器场景里，FIR 的行为稳定、相位关系更好理解，但阶数越高，延迟和计算量也越明显。

N 越大

参与平均或卷积的历史样本越多，输出越平滑；但延迟、RAM 占用和乘加次数都会上升。

FIR 的稳定性

FIR 不把历史输出反馈回来，所以不会因为反馈系数错误导致振荡发散，行为更可控。

滑动平均

它是最简单的 FIR。所有系数相等，容易实现，但频率选择性不如专门设计的低通 FIR。

窗函数低通

先由理想 sinc 得到无限长低通，再用有限窗截断，得到 MCU 可以运行的有限长度系数。

工程取舍

不要只追求曲线平。真正要看的是：动态慢了多少、控制是否还能接受、MCU 算力是否足够。

FIR 原理补充：为什么它有限？为什么它常说“线性相位”？

1. FIR 的公式到底在做什么？

FIR 的核心公式是：y[n] = Σ b[k]x[n-k]。 这句话翻成人话就是：拿最近 N 个输入样本，按一组固定权重加权求和。 只要 N 是有限的，一个输入样本最多只影响后面有限个输出点。

  • 滑动平均：所有权重都一样，适合理解和快速实现。
  • 低通 FIR：中间系数通常更大，两侧系数逐渐变小，频率选择性更好。
  • 底部的 b[k] 图，就是 FIR 的有限冲激响应。

2. FIR 为什么天然稳定？

因为 FIR 没有 y[n-1]、y[n-2] 这种历史输出反馈项。 输入数据走完这段有限长度的系数序列后，影响就结束了。 所以它不会像某些 IIR 那样因为反馈参数不当而发散。

  • 公式结构更直观；
  • 调参风险更低；
  • 适合对相位和波形形状有要求的场景。

3. 为什么 N 越大越慢？

对称 FIR 常见于低通设计。它的一个重要特点是群延迟近似固定： delay = (N-1) / (2Fs)。 例如 Fs=100Hz、N=21 时，群延迟约为 100ms。

  • N 越大，频率选择性通常越好；
  • N 越大，输出越稳，但响应越慢；
  • N 越大，每个采样周期需要做的乘加次数越多。

4. FIR 和 IIR 的直观区别

IIR 更像“带惯性的递推系统”，只保留少量历史输出也能形成很强平滑； FIR 更像“有限窗口内的加权平均”，用更多历史输入换稳定、可控、易分析的行为。

  • IIR：省计算、省 RAM，但反馈结构需要注意稳定性和相位。
  • FIR：更稳定、更可控，但阶数高时资源消耗明显。
  • MCU 工程里二者都常用，关键看实时性、噪声、延迟和算力。

5. 一个最实用的学习视角：把 FIR 当成“有限长度加权窗口”

看 FIR 不要只背公式。你可以把它理解成一个窗口：窗口滑过输入序列，窗口内每个样本乘以一个权重，累加后得到当前输出。 这个窗口越长，越能压掉高频乱跳，也越容易让真实变化变慢。

1

先定采样频率

Fs 决定每个样本间隔，也决定同样 N 下的真实时间延迟。

2

再定窗口长度 N

N 决定参与计算的历史样本数量，也决定 RAM 和乘加次数。

3

选择系数形状

简单平均适合快速平滑；窗函数低通适合更明确的频率目标。
