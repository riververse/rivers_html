1 · rivers_timer

小型 standalone SysTick timer module。默认自动配置 SysTick，SysTick 中断只推进系统时间；软件定时器到期判断和回调执行由主循环中的 poll 完成。

2 · rivers_queue

standalone mem / queue module。包含静态内存池分配器和固定单元环形队列；mem 动态效果和 queue 环形队列效果在本页中分开展示。

1

rivers_timer

rivers_timer 的核心定位是：用 Cortex-M SysTick 建立系统时间基准，提供 uptime、delay 和 ms 级软件定时器。它不使用 malloc/free，软件定时器使用静态数组，回调不在 SysTick 中断中执行。

1.1 · 支持功能

✓自动配置 Cortex-M SysTick，初始化时计算 reload，写入 LOAD / VAL / CTRL。
✓使用 SysTick 作为系统时基，固定打开 ENABLE、TICKINT、CLKSOURCE。
✓需要在 SysTick_Handler() 中调用 rivers_timer_tick_handler() 推进系统时间。
✓维护 32 位 us/ms uptime，并允许自然回绕。
✓维护 64 位 us/ms uptime，给软件定时器内部绝对到期时间比较使用。
✓us 转 ms 时保留 s_ms_remainder_us，避免长期丢弃不足 1ms 的误差。
✓提供 rivers_timer_get_tick()、rivers_timer_get_uptime_ms()、rivers_timer_get_uptime_us()。
✓提供 rivers_timer_delay_ms() 和 rivers_timer_delay_us() 忙等延时。
✓提供 ms 级软件定时器，支持单次和周期模式。
✓软件定时器来自静态数组 s_sw_timers[]，容量由 RIVERS_TIMER_MAX_SW_TIMERS 控制，默认 16。
✓不使用 malloc/free，不依赖 HAL、CMSIS 或原 rivers_osal。
✓软件定时器回调在 rivers_timer_poll() 中执行，不在 SysTick 中断中执行。

1.2 · 接口说明

接口作用典型调用位置阻塞 / 回调注意事项
rivers_timer_init()清零 uptime、清空软件定时器静态表、配置 SysTick。系统初始化。不阻塞，不执行用户回调。会配置 SysTick；若工程已有 SysTick 管理，需要统一。
rivers_timer_tick_handler()推进 us/ms uptime。SysTick_Handler()。不执行用户回调。只更新时间，不检测软件定时器是否到期。
rivers_timer_get_tick()返回 32 位 ms tick。普通/中断上下文。不阻塞，不回调。语义等同于 get_uptime_ms。
rivers_timer_get_uptime_ms()读取已累计的 32 位 ms uptime。时间戳、超时判断。不阻塞，不回调。不读取 SysTick VAL，只读累计变量。
rivers_timer_get_uptime_us()读取整 tick us，并结合 VAL 估算 tick 内 extra_us。短时间测量、us delay。不阻塞，不回调。返回 now_us + extra_us。
rivers_timer_delay_ms()基于 tick 差值忙等 ms。短暂等待。阻塞 CPU，不回调。不调用 HAL_Delay，不执行软件定时器回调。
rivers_timer_delay_us()基于 uptime_us 差值忙等 us。短暂硬件等待。阻塞 CPU，不回调。真正 us 周期任务应使用硬件定时器。
rivers_timer_poll()扫描软件定时器静态数组，处理到期定时器。while(1) 或任务态。可能执行用户回调。软件定时器到期判断发生在这里。
rivers_sw_timer_create()创建单次或周期软件定时器。初始化或业务创建阶段。不阻塞，不回调。回调函数不能为 NULL。
rivers_sw_timer_start()设置 expiry_ms = now_ms + period_ms 并置 active。启动定时器。不阻塞，不回调。timer_id 必须有效且已分配。
rivers_sw_timer_stop()停止定时器，保留 allocated。暂停定时器。不阻塞，不回调。可再次 start。
rivers_sw_timer_delete()删除定时器，清空槽位。不再需要定时器。不阻塞，不回调。delete 后 timer_id 失效。
rivers_sw_timer_set_period()修改周期/超时时长。动态调整定时器。不阻塞，不回调。active 时会重算下一次 expiry_ms。
rivers_sw_timer_set_remaining()修改 active 定时器的剩余时间。重设倒计时。不阻塞，不回调。未 active 返回 NOT_READY。

1.3 · 软件思维

中断只推进时间

SysTick 中断中调用 rivers_timer_tick_handler()，只累加 uptime，不做复杂业务，不检测软件定时器，不执行用户 callback。

poll 才处理软件定时器

rivers_timer_poll() 在主循环或任务态调用，负责遍历 s_sw_timers[]，判断 active timer 是否到期。

32 位对外，64 位内部

外部接口返回 32 位时间，允许自然回绕；软件定时器内部使用 64 位毫秒时间比较 expiry_ms。

delay 使用差值判断

delay 不重新配置 SysTick，而是记录 start，然后循环读取当前 uptime，用无符号差值判断是否达到目标。

us 时间 = 整 tick + 子节拍

get_uptime_us() 先读已累计整 tick us，再通过 SysTick VAL / COUNTFLAG 估算 tick 内 extra_us。

临界区保存/恢复 PRIMASK

进入临界区时保存 PRIMASK 并关中断，退出时根据原状态恢复，避免破坏外部已经关闭中断的状态。

软件定时器控制块包含 allocated、active、periodic、expiry_ms、period_ms、cb、arg。周期定时器到期后推进 expiry_ms，单次定时器到期后 active=false。

1.4 · 移植步骤

01

添加文件

把 rivers_timer.c 和 rivers_timer.h 加入工程，配置 include path。

02

配置时钟

设置 RIVERS_TIMER_CPU_CLOCK_HZ 和 RIVERS_TIMER_SYSTICK_CLOCK_HZ。

03

配置 tick

默认 RIVERS_TIMER_TICK_RATE_HZ = 1000UL，即 1ms tick。

04

接入 Handler

在 SysTick_Handler() 中调用 rivers_timer_tick_handler()。

#define RIVERS_TIMER_CPU_CLOCK_HZ      168000000UL
#define RIVERS_TIMER_SYSTICK_CLOCK_HZ  RIVERS_TIMER_CPU_CLOCK_HZ
#define RIVERS_TIMER_TICK_RATE_HZ      1000UL
#define RIVERS_TIMER_MAX_SW_TIMERS     16
void SysTick_Handler(void)
{
    rivers_timer_tick_handler();

    // 如果工程还依赖 HAL_Delay/HAL_GetTick，可保留：
    // HAL_IncTick();
}
如果工程中已有其他 SysTick 配置，注意 rivers_timer_init() 也会配置 SysTick，需要统一管理。

1.5 · 使用指南

只使用 uptime / tick

rivers_timer_init();

while (1) {
    uint32_t now_ms = rivers_timer_get_uptime_ms();
    uint32_t now_us = rivers_timer_get_uptime_us();
    (void)now_ms;
    (void)now_us;
}

delay_ms / delay_us

rivers_timer_delay_ms(5U);
rivers_timer_delay_us(500U);

周期软件定时器

static void blink_cb(void *arg)
{
    (void)arg;
    // 建议保持短小，避免阻塞 poll
}

int t = rivers_sw_timer_create(1000U, true, blink_cb, NULL);
if (t >= 0) {
    rivers_sw_timer_start(t);
}

while (1) {
    rivers_timer_poll();
}

单次定时器 / period / remaining / stop / delete

int once = rivers_sw_timer_create(500U, false, timeout_cb, NULL);
rivers_sw_timer_start(once);

rivers_sw_timer_set_period(once, 1000U);
rivers_sw_timer_set_remaining(once, 200U);

rivers_sw_timer_stop(once);
rivers_sw_timer_delete(once);
只使用 tick / delay / uptime，可以不调用 rivers_timer_poll()；使用软件定时器，必须周期性调用 rivers_timer_poll()。

rivers_queue

rivers_queue 包含两部分：rivers_mem 静态内存池分配器，以及 rivers_queue 固定单元环形消息队列。内存块链表和队列 ring buffer 是两套不同结构，下面会分开展示。

2.1 · 支持功能

✓rivers_mem 静态内存池，不使用系统 malloc/free。
✓默认内部静态堆，RIVERS_QUEUE_HEAP_SIZE 默认 4096U。
✓支持用户传入外部内存池。
✓支持内存对齐，默认 RIVERS_QUEUE_MEM_ALIGN_SIZE = 8U。
✓支持 rivers_mem_alloc/free、free size、min free size、largest free block 查询。
✓固定单元大小、固定单元数量的环形消息队列。
✓创建后 length 和 item_size 固定不变。
✓send/recv/from_isr 都是非阻塞立即尝试。
✓队列满不覆盖旧数据，队列空返回 EMPTY。
✓不提供 timeout、任务阻塞、等待链表、自动唤醒等 RTOS 语义。

2.2 · 接口说明

rivers_mem

接口作用输入 / 返回是否阻塞注意事项
rivers_mem_init()初始化静态堆。buffer,size / 无返回。不阻塞。NULL,0 使用内部默认堆；会重置内存池和活动队列链表。
rivers_mem_alloc()分配 payload。size / 指针或 NULL。不阻塞。返回地址按对齐粒度对齐。
rivers_mem_free()释放内存块。ptr / 无返回。不阻塞。通过 block header 找回块并尝试合并空闲块。
rivers_mem_get_free_size()查询所有空闲块总和。返回字节数。不阻塞。不代表一次最大可申请大小。
rivers_mem_get_min_free_size()查询历史最低剩余空间。返回字节数。不阻塞。用于观察内存峰值压力。
rivers_mem_get_largest_free_block()查询最大连续可申请 payload。返回字节数。不阻塞。用于观察碎片情况。

rivers_queue

接口作用输入 / 返回是否阻塞注意事项
rivers_queue_create()创建固定单元环形队列。length,item_size / 句柄或 NULL。不阻塞。控制块和 storage 都从 rivers_mem 分配。
rivers_queue_delete()删除队列。q / 无返回。不阻塞。释放 storage 和控制块，句柄失效。
rivers_queue_send()发送一个 item。q,item / 状态码。不阻塞。满了返回 FULL，不覆盖旧数据。
rivers_queue_recv()接收一个 item。q,item / 状态码。不阻塞。空了返回 EMPTY。
rivers_queue_send_from_isr()ISR 中立即尝试发送。q,item / 状态码。不阻塞。当前实现行为和 send 一致。
rivers_queue_recv_from_isr()ISR 中立即尝试接收。q,item / 状态码。不阻塞。当前实现行为和 recv 一致。
get_count/free_count查询已有元素/剩余可写数量。q / 数量。不阻塞。无效 q 返回 0。
get_length/item_size查询容量和单元大小。q / 数值。不阻塞。创建后固定。
rivers_queue_reset()清空队列状态。q / 状态码。不阻塞。只清 head/tail/count，不清 storage 旧内容。

2.3 · 软件思维

统一内存来源

内部默认堆和用户外部堆都由 rivers_mem_init() 决定，queue_create 不关心具体来源。

不透明句柄

队列结构体在 .c 内部定义，用户只拿 rivers_queue_t *，避免外部直接修改内部状态。

活动链表

创建时加入 s_queue_list，send/recv/delete 时检查句柄是否仍有效。

固定单元 ring buffer

head 是出队位置，tail 是入队位置，count 是当前元素数量。

拷贝语义

send 时 memcpy 到 tail，recv 时 memcpy 从 head 取出，队列保存 item 的内容副本。

非阻塞语义

from_isr 也只是立即尝试，不提供阻塞等待；满了 FULL，空了 EMPTY。

2.4 · 内存管理

默认内部静态堆

s_default_heap 是内部默认静态堆。RIVERS_QUEUE_HEAP_SIZE 默认 4096U。调用 rivers_mem_init(NULL, 0) 会使用它。

#ifndef RIVERS_QUEUE_HEAP_SIZE
#define RIVERS_QUEUE_HEAP_SIZE 4096U
#endif

rivers_mem_init(NULL, 0);

修改内部堆大小

// 方式一：包含头文件前定义
#define RIVERS_QUEUE_HEAP_SIZE 8192U
#include "rivers_queue.h"

// 方式二：编译选项
// -DRIVERS_QUEUE_HEAP_SIZE=8192U

用户自定义内存池

static uint8_t user_queue_heap[4096];

rivers_mem_init(user_queue_heap, sizeof(user_queue_heap));

具体占用估算

UART RX 队列 
typedef struct {
    uint8_t byte;
} uart_rx_msg_t;

#define UART_RX_QUEUE_LEN 128

s_uart_rx_queue = rivers_queue_create(
    UART_RX_QUEUE_LEN,
    sizeof(uart_rx_msg_t)
);

length = 128，item_size = 1，storage 约 128 字节；实际还要加 storage 分配块 header、队列控制块 sizeof(rivers_queue_t) 的分配块 header，以及对齐开销。

UART TX 队列 
typedef struct {
    uint16_t len;
    char text[64];
} uart_tx_msg_t;

#define UART_TX_QUEUE_LEN 8

s_uart_tx_queue = rivers_queue_create(
    UART_TX_QUEUE_LEN,
    sizeof(uart_tx_msg_t)
);

length = 8，item_size = sizeof(uart_tx_msg_t)，storage 约 8 * sizeof(uart_tx_msg_t)。由于结构体对齐，sizeof(uart_tx_msg_t) 可能约 66~68 字节，最终还要加 block header 和对齐开销。

rivers_mem_get_free_size() 是所有空闲块总和；rivers_mem_get_largest_free_block() 是最大连续可申请 payload。碎片存在时，前者大不代表一次能申请同样大的连续块。

2.5 · 移植步骤

01

添加文件

加入 rivers_queue.c 和 rivers_queue.h，配置 include path。

02

配置堆大小

根据队列数量和单元大小修改 RIVERS_QUEUE_HEAP_SIZE。

03

配置对齐

默认 RIVERS_QUEUE_MEM_ALIGN_SIZE = 8U，更严格平台可调大。

04

初始化内存池

创建队列前调用 rivers_mem_init()。

rivers_mem_init(NULL, 0);

rivers_queue_t *q = rivers_queue_create(128U, sizeof(uint8_t));
if (q == NULL) {
    while (1) {
        // 内存不足或参数错误
    }
}

非 Cortex-M 平台临界区覆盖

#define RIVERS_QUEUE_CRITICAL_ENTER()      my_critical_enter()
#define RIVERS_QUEUE_CRITICAL_EXIT(state)  my_critical_exit(state)
不要在队列运行过程中重新初始化 rivers_mem，否则会重置内存池并清空活动队列链表。

2.6 · 使用指南

// 创建字节队列
rivers_mem_init(NULL, 0);
rivers_queue_t *byte_q = rivers_queue_create(128U, sizeof(uint8_t));

// 创建结构体消息队列
typedef struct {
    uint16_t id;
    uint8_t data[8];
} app_msg_t;

rivers_queue_t *msg_q = rivers_queue_create(16U, sizeof(app_msg_t));
// send / recv
app_msg_t msg = { .id = 1 };

if (rivers_queue_send(msg_q, &msg) != RIVERS_QUEUE_OK) {
    // 队列满
}

while (rivers_queue_recv(msg_q, &msg) == RIVERS_QUEUE_OK) {
    // 处理 msg
}
// ISR 中快速入队，主循环出队
void UART_IRQHandler(void)
{
    uint8_t byte = read_uart_byte();
    (void)rivers_queue_send_from_isr(byte_q, &byte);
}

while (1) {
    uint8_t byte;
    while (rivers_queue_recv(byte_q, &byte) == RIVERS_QUEUE_OK) {
        // 主循环解析
    }
}
uint32_t count = rivers_queue_get_count(msg_q);
uint32_t free_count = rivers_queue_get_free_count(msg_q);

rivers_queue_reset(msg_q);
rivers_queue_delete(msg_q);
UART 示例核心：RX 队列按字节缓存；TX 队列保存字符串内容副本；中断只负责快速入队；主循环负责组帧/解析和通过 TX 队列打印；行缓存收到 \n 后回显；drop_count 观察队列满导致的丢弃。
